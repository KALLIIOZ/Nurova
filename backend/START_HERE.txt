================================================================================
                            🚀 INICIO RÁPIDO (5 MIN)
================================================================================

1. COPIAR ARCHIVO DE VARIABLES:
   $ cp .env.example .env

2. EDITAR .env Y AGREGAR TUS KEYS:
   GEMINI_API_KEY=tu_api_key_aqui
   GOOGLE_FORMS_API_KEY=tu_api_key_aqui
   SECRET_KEY=una_clave_secreta_larga
   
   Obtener keys en:
   - Gemini: https://makersuite.google.com/app/apikey
   - Google Forms: https://console.cloud.google.com

3. INSTALAR DEPENDENCIAS:
   $ python -m venv venv
   $ venv\Scripts\activate  (o source venv/bin/activate en Linux/Mac)
   $ pip install -r requirements.txt

4. INICIAR REDIS:
   En otra terminal: redis-server
   (Asegúrate que Redis está corriendo)

5. EJECUTAR 4 SERVICIOS EN TERMINALES DIFERENTES:
   Terminal 1: python run_service.py worker
   Terminal 2: python run_service.py psychologist
   Terminal 3: python run_service.py manager
   Terminal 4: python run_service.py gateway

6. VERIFICAR QUE FUNCIONA:
   Abre: http://localhost:8000/health
   Debe mostrar: {"status": "healthy", ...}

================================================================================
                        ⚡ COMANDOS ÚTILES DE DESARROLLO
================================================================================

Verificar que Redis está corriendo:
$ redis-cli ping

Limpiar base de datos (SQLite):
$ rm data/app.db

Ver logs de Docker:
$ docker-compose logs -f

Parar todo:
$ docker-compose down

Ejecutar con Docker (alternativa):
$ docker-compose up -d

================================================================================
                           ❓ SOLUCIÓN DE PROBLEMAS
================================================================================

❌ "Can't connect to Redis"
→ Asegúrate que redis-server (redis-cli ping)

❌ "ModuleNotFoundError: No module named 'fastapi'"
→ pip install -r requirements.txt

❌ "Port already in use"
→ Cambiar puerto en el comando: --port 8010

❌ "Database is locked"
→ Cerrar todos los servicios y reiniciar

❌ "CORS error from frontend"
→ Verificar BASE_URL en src/api/config.js
→ Debe ser: http://localhost:8000 (o IP correcta)
